## Tools and Frameworks

### LangChain: Introducing Rubrics — Agents that Evaluate and Correct Their Own Work
**Source**: LangChain Blog | **Date**: 25 Jun 2026 | **Signal**: 7.5 | **Tags**: Evaluation, Agent Engineering

LangChain's RubricMiddleware adds a self-evaluation loop to agent runs: configure a rubric (a structured criterion set), choose a grader model, and the agent checks its own outputs against the rubric before returning results — rerunning steps that fail the grade. This implements inference-time self-correction at the pipeline level rather than requiring it at the model level, meaning it works with any model. The practical pattern is to write rubrics for the failure modes you've observed in production: if your agent frequently produces outputs that pass syntactic checks but fail business logic requirements, a rubric expressing those requirements gets enforced automatically. This is a direct operationalization of the "critique and revise" prompting pattern as configurable middleware rather than ad-hoc prompt engineering.

[Source](https://www.langchain.com/blog/deep-agents-rubrics)

### LangChain: Introducing LangSmith Engine — Automated failure clustering and fix proposals
**Source**: LangChain Blog | **Date**: 25 Jun 2026 | **Signal**: 7.6 | **Tags**: MLOps, Evaluation

LangSmith Engine watches production traces, clusters failures into named issues, and proposes targeted fixes — closing the loop between agent monitoring and agent improvement without requiring manual trace review. The clustering step is architecturally important: rather than presenting engineers with raw trace logs, the engine groups failures by inferred root cause, reducing the cognitive overhead of debugging agentic pipelines where failures may be distributed across dozens of tool calls. For teams running agents in production today without systematic failure analysis, LangSmith Engine provides the observability-to-improvement cycle that currently requires manual triage — an LLMOps pattern analogous to error aggregation tools (Sentry, Datadog) but designed for the structure of agent execution traces.

[Source](https://www.langchain.com/blog/langsmith-engine)
