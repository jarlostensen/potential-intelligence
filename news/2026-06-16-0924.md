## Claude and Anthropic

### Fable 5 "Jailbreak" Confirmed: Asking the Model to Fix Deliberately Vulnerable Code
**Source**: Simon Willison | **Date**: 16 Jun 2026 | **Signal**: 7.8 | **Tags**: Security, Anthropic

Cybersecurity expert Katie Moussouris — who reviewed the White House report on the Fable 5 jailbreak at Anthropic's request — confirmed to Simon Willison that the triggering "jailbreak" was IT professionals asking Fable to help find and patch bugs in code that had deliberate vulnerabilities planted alongside known CVEs. This is a significant clarification: the export control was not triggered by the model autonomously discovering vulnerabilities or by an adversarial exploit — it was triggered by a routine security-review workflow that any senior security engineer or developer might run. For practitioners, this changes the risk model substantially: the question is no longer "is the model capable of weaponisable hacking?" but "is any AI-assisted code-security review workflow now potentially subject to export controls?" The answer appears to be yes, at least for the most capable frontier models.

### GitHub's Official MCP Server
**Source**: GitHub | **Date**: 16 Jun 2026 | **Signal**: 7.2 | **Tags**: MCP, Claude Code

GitHub has released an official MCP server (30K stars, github/github-mcp-server) that exposes GitHub APIs — repos, issues, PRs, code search, Actions — as MCP tools consumable by Claude Code, Cursor, Windsurf, and any other MCP-capable coding agent. This is the highest-value MCP integration to date from a first-party vendor: every engineering team already has GitHub as infrastructure, and making that state directly addressable by a coding agent without custom integration code is immediately useful. Practitioners should add this to their Claude Code setup now and evaluate which GitHub workflows (PR drafting, issue triage, CI investigation) benefit most from direct agent access.

## LLM Models and Capabilities

### OpenMythos: RLVR-Fine-Tuned Open-Source LLM for Cybersecurity
**Source**: r/LocalLLaMA | **Date**: 15 Jun 2026 | **Signal**: 7.5 | **Tags**: LLM Models, Security, AI-Assisted Dev

Built for the Build Small Hackathon, OpenMythos is an open-source LLM fine-tuned specifically for cybersecurity tasks using RLVR (Reinforcement Learning from Verifiable Rewards). The team's motivation is directly relevant: general-purpose LLMs hallucinate CVE details and miss real vulnerability patterns. Their RLVR setup uses verifiable security outputs (does the generated code actually fix the CVE? does the analysis correctly identify the vulnerability class?) as the reward signal, making this one of the more technically rigorous domain-specific fine-tuning approaches published openly. For practitioners building security tooling on LLMs, or for anyone interested in domain-specific RLVR training, the training methodology writeup is the primary value here — the model itself may be less competitive than frontier alternatives, but the approach is directly replicable.

[Source](https://www.reddit.com/r/LocalLLaMA/comments/1u786se/)

## Agent Engineering

### CopilotKit: Frontend Stack for Agents and the AG-UI Protocol
**Source**: GitHub | **Date**: 16 Jun 2026 | **Signal**: 7.0 | **Tags**: Agent Engineering, Tools

CopilotKit (35K stars) is the frontend SDK for building agent-native UIs — React, Angular, mobile, and Slack integrations — and is the primary driver of the AG-UI Protocol, an emerging standard for bidirectional communication between AI agents and user interfaces. AG-UI solves a problem that pure backend agent frameworks ignore: how does an agent stream intermediate state, request user approval, or display structured output to a frontend without custom engineering for every deployment? The protocol standardises this handshake. For full-stack teams building agent applications, CopilotKit is the practical implementation path for AG-UI, and the protocol itself is worth tracking as it may become a cross-framework standard.

[Source](https://github.com/CopilotKit/CopilotKit)
